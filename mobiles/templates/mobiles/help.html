Important note on unscribing and block lists

It is critical that individuals can block their mobiles from receiving unwanted text messages.

This is enforced by our SMS provider (currently Twilio). If you send a text to STOP, UNSUBSCRIBE, CANCEL, END or QUIT, then no further messages will be sent to your phone. Textivist will receive notice of this, but has no ability to override it.

There is also a STOPALL command - it might be nice to apply this to all a given users subscriptions. Currently, Twilio only applies the block per endpoint.

You can send a START, YES or UNSTOP text to start receiving messages again.

SMS Commands

  • REGISTER: Will create new Textivist account if not present, and add membership to endpoint's org.
  • SET (Firstname Lastname): Registers your name to the mobile
  • SET OWNER (Firstname Lastname): Registers your name to the mobile
  • SET POSTCODE (Your Postcode): Registers your postcode to the mobile (not yet implemented)
  • ADD (TOPIC): Subscribes to given topic shortcode
  • MUTE (TOPIC): Stops subscription to given topic shortcode
  • CAST (TOPIC) (Message Text): Broadcasts your Message Text to all topic subscribers
